# Data Broker Policy

## Overview

This folder contains the repository’s formal policy materials regarding data broker access, commercial reuse, identity-linked exploitation, derivative profiling, and compensation requirements.

These materials are published as part of the `broker-access-and-compensation-policy` branch and are intended to establish clear notice that this repository is not a free commercial intake source for brokers, aggregators, people-search platforms, profile builders, commercial AI dossier systems, identity-resolution vendors, or derivative monetization services.

This repository is tied to the verified academic, authorship-linked, and identity-linked public record of Justin-Ames Gamache. As a result, unauthorized commercial reuse is not treated as a neutral copying event. It is treated as a matter involving identity, authorship, record integrity, reputational framing, and non-consensual commercial exploitation.

## Core Position

This repository may be publicly viewable, but public visibility does **not** create:

- implied consent;
- broker reuse rights;
- permission to scrape for commercial purposes;
- permission to ingest repository content into people-search or profile systems;
- permission to enrich, score, stitch, rank, repackage, or monetize identity-linked record material;
- or permission to convert this repository into a commercial subject asset.

Any commercial, broker, profiling, enrichment, republication, or derivative use involving the identity-linked record of Justin-Ames Gamache requires prior written authorization and compensation.

## Purpose of This Folder

This folder exists to do four things:

1. provide explicit notice of non-consent to broker profiling and commercial exploitation;
2. establish that no commercial license is granted by public access alone;
3. define the compensation and authorization requirements for any commercial actor seeking reuse; and
4. preserve the integrity, authorship, chronology, and contextual meaning of a curated verified record.

## Identity and Record Integrity

These policy materials are grounded in the position that the repository is a curated, identity-linked, authorship-linked, and integrity-protected archive associated with Justin-Ames Gamache.

Unauthorized downstream extraction, enrichment, scoring, and derivative republication create a substantial risk of:

- identity misuse;
- record distortion;
- reputational misframing;
- authorship decontextualization;
- stale or misleading derivative profiles;
- and exploitative subject-based monetization.

This folder exists in part to eliminate any plausible claim that such conduct occurred without notice.

## No Implied Consent

No part of this folder, this branch, or this repository should be interpreted as granting consent for:

- data broker harvesting;
- commercial people-search indexing;
- subject profiling;
- identity enrichment;
- dossier creation;
- commercial scoring or ranking;
- derivative profile generation;
- resale or relicensing of extracted repository material;
- or automated commercial exploitation of identity-linked record content.

Silence is not consent. Public access is not a license. Machine readability is not permission.

## Commercial Actors

Any commercial actor seeking to use repository content for any broker, profiling, enrichment, aggregation, or monetized derivative purpose must:

- identify itself fully;
- disclose the intended use;
- disclose any downstream recipients or affiliated entities;
- obtain express written authorization; and
- negotiate compensation before any such use occurs.

Until then, the answer is no.

## Final Position

This repository is public for review, documentation, and record integrity.

It is not public for broker capture, profile monetization, or commercial identity exploitation.

If a commercial actor wants to use this repository as a revenue-producing asset, that actor must ask first and pay for the privilege.
