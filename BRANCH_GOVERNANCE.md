# FIFTH-AMENDMENT-INSTALLMENT
## Branch Governance Code
**Prepared by:** FIFTH AMENDMENT

## Purpose

This branch is a restricted governance branch dedicated to the preservation, interpretation, and verification of Vermont-related materials framed as advocacy work, record-provenance disputes, expungement-context corrections, and rights-protective public record architecture.

It is not a casual editing branch.
It is not a rumor repository.
It is not an open narrative sandbox.

## Controlling Rule

No file in this branch may be altered, replaced, rewritten, renamed, force-pushed, or deleted except through a signed, reviewable, hash-verified change that preserves historical continuity and explains the reason for alteration in a permanent release note.

## Vermont Framing Rule

All Vermont-related materials in this branch shall be framed as advocacy work, public-interest concern, record-governance conflict, expungement-context interpretation, or rights-protective activity.

They shall not be framed as free-floating character evidence, identity shorthand, gossip residue, or context-deprived accusation material.

## Source Priority Rule

Priority shall be given in the following order:

1. Signed court orders and docket materials
2. Repository-controlled provenance files
3. Archived copies that preserve update history or expungement context
4. Public reporting only when used with contextual qualification
5. Scraped, aggregated, brokered, or context-deficient material — non-controlling

## Restrictive Edit Rule

A proposed change is invalid unless it includes all of the following:

- a signed commit
- a release-note entry
- updated SHA256 checksum output
- a short reason for change
- a statement confirming whether the change is corrective, additive, or structural
- no deletion of prior provenance without archival retention

## No Silent Rewrite Rule

No silent rewrite, stealth cleanup, quiet replacement, force push, or undocumented normalization is permitted in this branch.

Any attempt to revise interpretive language, source hierarchy, legal framing, or rights notices without explicit record of change is non-canonical.

## Historical Preservation Rule

Where a file is superseded, the prior version must remain archived or tagged so that provenance, chronology, and interpretive history remain reviewable.

## Integrity Rule

Where authenticity, chronology, or completeness is disputed, the controlling version is the one supported by:

- signed tag or signed commit history
- matching release notes
- checksum continuity
- mirrored archival copy
- repository governance records

## Branch Status

This branch is intended to function as a restricted constitutional-governance installment and not as a general collaboration surface.
