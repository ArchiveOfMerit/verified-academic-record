# Contributing to The Archive of Merit Project

Thank you for your interest in contributing to this repository.

The Archive of Merit Project is a structured archival and documentation project focused on preserving, organizing, and presenting verified records. Contributions must support record integrity, clarity, and responsible handling of source material.

## Before Contributing

Please review:

- `README.md`
- `CODE_OF_CONDUCT.md`
- any branch-specific documentation or repository notes

By contributing, you agree to follow this repository’s standards for professionalism, accuracy, and respectful conduct.

## What Contributions Are Appropriate

Contributions may include:

- correcting factual, typographical, formatting, or metadata errors
- improving citations, dates, source labels, or file organization
- clarifying provenance or record structure
- proposing documentation improvements that strengthen archival usability
- submitting clearly identified supplemental materials relevant to the repository’s scope

## What Is Not Appropriate

Do not use this repository to:

- submit knowingly false, misleading, or unverified claims
- post harassment, defamatory content, or personal attacks
- publish private personal information without clear justification and permission where required
- disrupt archival structure with irrelevant, duplicative, or bad-faith submissions
- alter provenance, dates, authorship markers, or metadata without explanation
- treat the repository as a debate forum, grievance dump, or trolling venue

## Contribution Standards

All contributions should be:

- relevant to the repository’s purpose
- accurate and supported where appropriate
- clearly labeled if they are derived, summarized, reconstructed, or interpretive
- limited in scope and easy to review
- respectful of privacy, safety, and archival integrity

Where uncertainty exists, state it directly. Do not conceal ambiguity behind confident wording.

## Pull Requests

When opening a pull request, please:

1. describe the proposed change clearly
2. explain why the change is necessary or beneficial
3. identify the affected files, branches, or record areas
4. note any source basis, provenance issue, or documentation rationale
5. keep unrelated changes out of the same pull request

Small, precise pull requests are preferred over large mixed revisions.

## Issues

Issues may be used to report:

- factual errors
- broken links
- metadata problems
- organizational concerns
- documentation gaps
- record-integrity concerns

Issues should remain specific, evidence-oriented, and tied to repository content.

## Branch and Record Discipline

This repository may contain branches serving different archival or operational purposes. Contributors must respect branch-specific conventions, including naming, metadata, and preparation markers.

Do not move material across branches or rewrite branch context without explicit justification.

## Review and Acceptance

All submissions are reviewed at maintainer discretion.

Submission does not guarantee acceptance. Contributions may be edited, declined, closed, or removed if they do not fit the repository’s purpose, standards, or structural conventions.

## Questions or Concerns

For contribution-related questions or conduct concerns, use the reporting or contact method identified in `CODE_OF_CONDUCT.md`.
