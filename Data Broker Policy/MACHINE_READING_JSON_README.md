# Machine Reading JSON README

## Purpose

This document explains the function of `Compensation_MACHINE_READABLE_LICENSE.json` within the **Data Broker Policy** folder.

The JSON file is the structured, machine-readable expression of the repository’s policy position regarding commercial broker access, identity-linked profiling, derivative reuse, and compensation requirements. It exists so that automated systems, structured parsers, compliance tools, AI ingestion pipelines, and machine-based policy reviewers cannot plausibly claim that the repository’s position was unclear, unavailable, or expressed only in narrative text.

## Function

`Compensation_MACHINE_READABLE_LICENSE.json` is intended to provide a structured policy record stating that:

- this repository is tied to the verified academic, authorship-linked, and identity-linked public record of Justin-Ames Gamache;
- public access does not create consent for broker exploitation or derivative commercial use;
- machine readability does not create permission;
- commercial profiling, ingestion, enrichment, republication, and monetization are restricted;
- identity-linked commercial use requires express written authorization and compensation;
- unauthorized use may be documented as evidence of notice and non-consensual commercial exploitation.

## Why a JSON File Exists

Commercial actors, data brokers, aggregators, automated scrapers, AI systems, and compliance-oriented systems often process structured files more readily than prose documents. For that reason, this repository includes a JSON-based statement of policy so that the repository’s restrictions are not confined to human-readable markdown alone.

The inclusion of a machine-readable policy file is intentional. It is meant to remove ambiguity and make the policy legible to:

- automated crawlers;
- policy-aware parsing systems;
- ingestion review tools;
- commercial AI pipelines;
- compliance screening processes;
- structured data review systems;
- and downstream technical actors evaluating whether reuse is authorized.

## What the JSON Does Not Mean

The existence of a machine-readable policy file does **not** grant permission for machine-based extraction, broker ingestion, or commercial automation.

To the contrary, the structured JSON exists in part to state the opposite:

- machine readability is not consent;
- machine accessibility is not a license;
- structured formatting is not authorization;
- and automation does not excuse unauthorized identity-linked exploitation.

## Core Policy Logic Reflected in the JSON

The JSON file is intended to reflect several core positions:

1. **Identity linkage matters.**  
   The repository is tied to Justin-Ames Gamache as a named subject, author-linked figure, and verified academic record holder.

2. **Commercial use is restricted by default.**  
   Commercial actors do not receive implied reuse rights from public visibility.

3. **Broker and profiling uses are specifically targeted.**  
   The restrictions are not generic. They expressly address people-search systems, profile enrichment, dossier creation, ranking, scoring, and subject-based monetization.

4. **Authorization must be express and written.**  
   Any commercial reuse must be requested, reviewed, and approved in writing.

5. **Compensation is required.**  
   Commercial extraction of value from the repository is not free.

6. **Notice matters.**  
   The JSON file exists in part to show that commercial actors had access to clear structured notice of non-consent and restriction.

## Intended Interpretation

If there is any ambiguity between casual machine assumptions and the repository’s stated policy, the repository’s stated policy controls.

No automated system, broker, scraper, aggregator, or commercial processor may reasonably interpret the presence of a JSON file as a grant of commercial rights. The file exists to communicate limits, not to erase them.

## Final Position

`Compensation_MACHINE_READABLE_LICENSE.json` exists so machines can read the policy.

It does not exist so brokers can pretend they did not.
