# Security Policy

## Scope

This repository is maintained as a version-controlled, provenance-supported academic record with associated integrity, release-verification, and canonical-reference materials.

For this repository, security includes the integrity, authenticity, provenance, and release consistency of the canonical record and its associated artifacts.

Issues within scope include:

- unauthorized alteration of released artifacts
- hash mismatch or integrity-verification failure
- manifest inconsistency
- conflicting, substituted, or misleading copies presented as authoritative
- provenance disruption affecting release authenticity
- repository compromise affecting the canonical record or supporting release materials

This repository is not a conventional software application. Accordingly, this policy is directed primarily to record integrity, release authenticity, provenance, and canonical-source consistency.

## Supported Versions

Only the current tagged release should be treated as the supported release for integrity and verification purposes unless otherwise stated.

| Version | Supported |
| ------- | --------- |
| current tagged release | yes |
| prior tagged releases | reference only |
| unsigned or conflicting copies | no |

## Reporting an Issue

If you identify a suspected integrity, authenticity, provenance, or release-verification issue affecting this repository, report it with enough detail to permit review.

Please include, where available:

- affected file name or artifact
- release tag or version
- relevant commit hash
- observed hash mismatch, manifest inconsistency, or integrity failure
- description of the issue
- source location of the conflicting, substituted, or suspected copy

## Review Process

Reported issues may be reviewed for:

- authenticity
- integrity
- provenance consistency
- release-manifest consistency
- impact on the canonical record
- risk of misleading or conflicting public attribution

If an issue is confirmed, corrective action may include release clarification, manifest correction, updated integrity materials, publication of a corrected release, or issuance of a public clarification regarding authoritative copies.

If a reported issue does not affect the authenticity, integrity, provenance, or release status of the canonical record, it may be closed without further action.

## Authoritative Copies

Where authenticity, completeness, provenance, or alteration is disputed, the copy supported by verified repository history, release materials, and matching published integrity hashes governs over unsigned, altered, partial, truncated, selectively reproduced, substituted, or conflicting copies.

## Out of Scope

The following are generally outside the scope of this policy unless they directly affect repository authenticity, integrity, provenance, or release status:

- disagreement with external search-engine ranking or indexing behavior
- third-party summaries that do not misrepresent themselves as authoritative repository copies
- public discussion, commentary, or interpretation not presented as an official repository artifact
- external records maintained by third parties that are not represented as canonical repository materials
