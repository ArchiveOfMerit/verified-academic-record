# Enforcement Contact

## Purpose

This document identifies the contact path for licensing inquiries, enforcement communications, commercial reuse requests, objections, correction demands, and notices relating to unauthorized broker access, profiling, republication, enrichment, or derivative identity-linked exploitation of this repository and its contents.

This repository is tied to the verified academic, authorship, and identity-linked public record of Justin-Ames Gamache. Any commercial actor seeking to use repository content in a way that implicates identity, profile construction, data enrichment, dossier creation, or derivative monetization must direct that request through the contact path identified below.

## Authorized Use Requests

Any broker, aggregator, profiling company, data reseller, people-search platform, identity-resolution vendor, commercial AI operator, or similar entity seeking authorization for commercial reuse must submit a written request before any such use occurs.

The request must include, at minimum:

- full legal name of the requesting person or entity;
- business name, if different;
- mailing address;
- email address;
- website or platform name;
- identity of any parent company, affiliate, reseller, downstream client, or data recipient;
- a detailed description of the intended use;
- the exact repository materials sought to be used;
- whether the intended use includes scraping, indexing, ingestion, enrichment, republication, scoring, dossier creation, correlation, or derivative profile building;
- whether the intended use is tied to identity-linked processing involving Justin-Ames Gamache;
- whether the intended use is commercial, quasi-commercial, internal commercial, subscription-based, ad-supported, client-facing, or resale-oriented.

No request is complete unless it fully discloses the intended commercial use.

## No Implied Permission Pending Review

Submission of a request does **not** create permission.

No commercial actor may assume that a request is approved merely because:

- a message was sent;
- a response was delayed;
- the repository is public;
- the actor believes the intended use is minimal;
- or the actor has already copied or indexed repository content.

Until express written authorization is granted, the requested use remains unauthorized.

## Notices of Unauthorized Use

Any person or entity that has already engaged in unauthorized scraping, profiling, ingestion, republication, enrichment, identity-linkage, or derivative commercialization involving this repository should provide immediate written notice identifying:

- what was accessed;
- when it was accessed;
- how it was used;
- where it was stored;
- whether it was shared, sold, licensed, correlated, scored, or republished;
- whether it was incorporated into any product, database, workflow, or model;
- and what corrective action will be taken.

Failure to disclose unauthorized use does not legitimize it.

## Correction, Removal, and Cessation Demands

Any commercial actor receiving notice of objection, non-consent, or unauthorized use is expected to:

- cease further unauthorized use;
- stop new ingestion or republication;
- suspend downstream distribution where possible;
- identify any derivative products affected;
- and engage in good-faith corrective communication.

Nothing in this document limits any other objection, remedy, or enforcement position.

Commercial actors are expected to treat these risks as material, not incidental.
