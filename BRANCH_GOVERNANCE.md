# FIFTH-AMENDMENT-INSTALLMENT
## Branch Governance Code
**Prepared by:** FIFTH AMENDMENT  
**Version:** 2.0.0  
**Status:** Restricted Governance Branch Directive  
**Last Verified:** April 12, 2026

## Purpose

This branch is a restricted governance branch dedicated to the preservation, interpretation, verification, and controlled monitoring of Vermont-related materials framed as advocacy work, record-provenance disputes, expungement-context corrections, rights-protective public record architecture, and search-surface integrity review.

It is not a casual editing branch.  
It is not a rumor repository.  
It is not an open narrative sandbox.  
It is not a freeform storage location for decontextualized legal-index, search-engine, or public-accusation fragments.

## Controlling Rule

No file in this branch may be altered, replaced, rewritten, renamed, force-pushed, deleted, or structurally relocated except through a signed, reviewable, hash-verified change that preserves historical continuity, explains the reason for alteration in a permanent release note, and states whether the change is corrective, additive, structural, or monitoring-related.

## Vermont Framing Rule

All Vermont-related materials in this branch shall be framed as advocacy work, public-interest concern, record-governance conflict, expungement-context interpretation, search-surface correction, or rights-protective activity.

They shall not be framed as free-floating character evidence, identity shorthand, gossip residue, context-deprived accusation material, or algorithmically amplified shorthand detached from record context.

## Source Priority Rule

Priority shall be given in the following order:

1. Signed court orders and docket materials  
2. Repository-controlled provenance files and governance records  
3. Archived copies preserving chronology, update history, or expungement context  
4. DOI-linked or repository-governed interpretive materials  
5. Public reporting only when used with explicit contextual qualification  
6. Search-engine summaries, AI summaries, or legal-index excerpts only for monitoring and contradiction tracking, never as controlling authority  
7. Scraped, aggregated, brokered, or context-deficient material — non-controlling

## Search Monitoring Rule

Any Vermont-related search query, legal index, AI summary, public snippet, or external search surface that materially affects interpretation of branch materials must be logged, reviewable, and handled as a monitored external surface rather than a governing source.

Elevated review applies in particular to query forms and case-caption forms that increase the likelihood of surfacing stale, fragmented, or non-authoritative materials.

Such materials may be preserved for evidentiary, monitoring, correction, or comparison purposes, but they do not control branch interpretation.

## Restrictive Edit Rule

A proposed change is invalid unless it includes all of the following:

- a signed commit  
- a release-note entry  
- updated SHA256 checksum output where applicable  
- a short reason for change  
- a statement confirming whether the change is corrective, additive, structural, or monitoring-related  
- no deletion of prior provenance without archival retention  
- confirmation that the change does not demote controlling source priority without express notation

## No Silent Rewrite Rule

No silent rewrite, stealth cleanup, quiet replacement, force push, undocumented normalization, concealed reframing, or unlogged search-monitor adjustment is permitted in this branch.

Any attempt to revise interpretive language, source hierarchy, legal framing, rights notices, search-monitoring logic, or public-facing contextual safeguards without explicit record of change is non-canonical.

## Historical Preservation Rule

Where a file is superseded, the prior version must remain archived, tagged, or otherwise retained so that provenance, chronology, interpretive history, and monitoring history remain reviewable.

Where an external surface changes over time, the branch should preserve enough documentation to distinguish original context, later modification, and current interpretive status.

## Integrity Rule

Where authenticity, chronology, completeness, or interpretive authority is disputed, the controlling version is the one supported by:

- signed tag or signed commit history  
- matching release notes  
- checksum continuity  
- mirrored archival copy  
- repository governance records  
- preserved provenance of external-reference comparison where relevant

## External Surface Rule

External search results, legal-index pages, news fragments, broker pages, and AI-generated summaries may be recorded in this branch only for one or more of the following purposes:

- contradiction tracking  
- provenance comparison  
- decontextualization analysis  
- reputational harm monitoring  
- expungement-context correction  
- search-surface governance documentation

They shall not be imported or preserved as if they are self-authenticating, self-interpreting, or controlling authority.

## Branch Status

This branch is intended to function as a restricted constitutional-governance installment, provenance-preservation surface, and monitored rights-protective archive, not as a general collaboration surface.
